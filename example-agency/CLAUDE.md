# example-agency — правила проекта

> Этот файл загружается автоматически, когда Claude Code работает внутри папки `example-agency/`. Он дополняет корневой `CLAUDE.md`, а не заменяет его.

## Контекст

**Что это:** клиентский проект агентства. Здесь живут сделки, встречи, стратегия, контент.

**Кто пользователь в этом контексте:** сооснователь / продавец / проект-менеджер. Работает с клиентами, выставляет КП, ведёт сделки от лида до закрытия.

## Структура (кратко)

- `CRM/Active/` — 1 папка = 1 сделка. Главный файл — `deal.md`
- `CRM/Archive/` — закрытые сделки (won + lost)
- `CRM/Prospects/` — потенциальные до первой встречи
- `Docs/Strategy/` — стратегия, гипотезы, лидген
- `Docs/Tasks/tasks.md` — командные задачи
- `Client-Meetings/` — транскрипты и саммари встреч

## Правила работы со сделками

### 1. Создание новой сделки

**НИКОГДА** не создавай папку сделки вручную без шаблона. Используй `/project-context-setup` или скопируй структуру существующей сделки.

Шаблон имени: `YYYY-MM-DD_название-клиента/` (дата — первый контакт, slug kebab-case).

**Обязательные файлы в новой сделке:**
- `deal.md` — статус, стоимость, этапы, контакты ЛПР, следующие шаги
- `changelog.md` — хронология (append-only, последние записи сверху)

### 2. Обновление deal.md

**ВСЕГДА** при любом изменении по сделке:
1. Обнови соответствующее поле в `deal.md` (статус, стоимость, даты)
2. Добавь запись в `changelog.md` (YYYY-MM-DD + что изменилось + почему)

**НЕ** перезаписывай `deal.md` целиком. Точечные правки через `Edit`.

### 3. Работа с встречами

Транскрипт встречи с клиентом:
1. Положи сырой файл в `Client-Meetings/YYYY-MM-DD_клиент_тип.md`
2. Прогони через `/call-analysis` — получишь action items
3. Скопируй саммари + action items в `CRM/Active/<сделка>/attachments/` и обнови `deal.md`
4. **НЕ** держи транскрипт только в `Client-Meetings/` — он должен быть связан со сделкой

### 4. Финансовые данные

- Суммы, бюджеты, маржинальность — только в `deal.md` и `Docs/Strategy/`
- **НИКОГДА** не выводи финансовые данные в чат без явного запроса
- При работе с финансами — двойная проверка перед коммитом

## Правила работы с INBOX проекта

Если пользователь кладёт файл в `example-agency/INBOX/`:

1. Прочитай файл (хотя бы начало)
2. Определи, к какой сделке относится
3. Если есть существующая сделка → переместить в `CRM/Active/<deal>/attachments/`
4. Если сделки ещё нет → создать в `CRM/Prospects/` и переместить туда
5. Если файл — транскрипт встречи → `Client-Meetings/` + связать со сделкой

**НЕ** оставляй файл в INBOX после обработки. **НЕ** копируй — перемещай (`mv`).

## Типовые команды для этого проекта

| Команда | Когда использовать |
|---------|-------------------|
| `/prospect-research <компания> <url>` | Новый холодный лид — нужен полный анализ |
| `/prospect-brief <компания>` | Нужен внешний документ для партнёра/клиента |
| `/call-analysis` | Разбор транскрипта встречи |
| `/call-quality-review` | Оценка качества встречи (для ретро продажника) |
| `/create-proposal` | Генерация КП в .docx |
| `/deep-research` | Ресёрч по индустрии клиента |

## Чего НЕ делать

- **НЕ** копируй данные клиентов между сделками
- **НЕ** обсуждай одного клиента в контексте другого
- **НЕ** создавай публичные кейсы из сделок под NDA (сверяйся с `deal.md` — есть ли пометка NDA)
- **НЕ** путай `CRM/Active/` и `CRM/Prospects/` — разный уровень зрелости лида
