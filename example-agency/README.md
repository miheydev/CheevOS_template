# example-agency

> Скелет **клиентского проекта** (агентство, консалтинг, фриланс, продажная команда). Показывает, как раскладывать CRM-сделки, встречи, стратегические документы и контент.
>
> **Это пример.** Переименуй папку (`example-agency` → `my-agency`), отредактируй `CLAUDE.md` под себя, начни работать.

## Структура

```
example-agency/
├── CLAUDE.md               # Правила проекта (для AI)
├── README.md               # Этот файл
│
├── CRM/
│   ├── Active/             # Активные сделки (1 папка = 1 сделка)
│   ├── Archive/            # Завершённые сделки (выигранные + проигранные)
│   └── Prospects/          # Потенциальные клиенты до первой встречи
│
├── Docs/
│   ├── Strategy/           # Стратегия агентства, гипотезы, планы
│   ├── Tasks/              # Командные задачи (tasks.md)
│   ├── Content/
│   │   ├── Drafts/         # Черновики постов/статей
│   │   └── Published/      # Опубликованное
│   └── Research/           # Ресёрчи по индустриям, клиентам, технологиям
│
├── Client-Meetings/        # Транскрипты и саммари встреч с клиентами
└── INBOX/                  # Входящие файлы для сортировки (локальный inbox проекта)
```

## Работа со сделками (CRM)

Каждая сделка — отдельная папка в `CRM/Active/` с шаблоном имени:

```
CRM/Active/YYYY-MM-DD_название-клиента/
├── deal.md                 # Главный файл — статус, стоимость, этапы, контакты, следующие шаги
├── changelog.md            # Хронология изменений по сделке
├── business-analysis.md    # Внутренний анализ (из /prospect-research)
├── briefing-for-partner.md # Внешний брифинг (из /prospect-brief)
└── attachments/            # PDF, документы, скриншоты
```

**Типовые команды для работы:**
- `/prospect-research <компания> <url>` — полный анализ потенциального клиента (5 параллельных агентов)
- `/prospect-brief <компания>` — внешний брифинг на основе внутреннего анализа
- `/create-proposal` — генерация КП в .docx
- `/call-analysis` / `/call-quality-review` — разбор транскрипта встречи

## Workflow: от холодного лида к сделке

```
1. Холодный лид появился
   ↓
2. /prospect-research <компания> <url>
   ↓ создаёт: CRM/Prospects/YYYY-MM-DD_клиент/business-analysis.md
   ↓
3. Первая встреча назначена
   ↓ переместить: CRM/Prospects/ → CRM/Active/
   ↓ /prospect-brief — внешний документ для клиента
   ↓
4. Встреча прошла → транскрипт в Client-Meetings/
   ↓ /call-analysis — саммари + action items
   ↓ обновить deal.md + changelog.md
   ↓
5. Сделка закрыта (won/lost)
   ↓ переместить: CRM/Active/ → CRM/Archive/
```

## Что положить в INBOX проекта

Файлы, которые точно относятся к этому проекту, но ещё не распределены по папкам сделок:
- Скриншоты переписки с клиентом
- Договоры, НДА, акты
- Брифы от клиента
- Технические спецификации

Разбирается скиллом `/inbox-root-processing` (с учётом проектных правил из `CLAUDE.md`).
