# Root INBOX Processing

> **Skill Type**: On-Demand
> **Trigger**: "разбери INBOX", "что в INBOX", "возьми из INBOX"

---

## Когда использовать

**НЕ проверяй INBOX автоматически** — только когда пользователь попросит.
**Главное правило:** если взял файл из INBOX — после обработки перемести его в нужное место, не клади обратно.

---

## Workflow

### Step 1: Проверить содержимое

```bash
ls INBOX/
```

Если только `README.md` — INBOX чист, продолжай работу.

### Step 2: Определить проект для каждого файла

Маршрутизация зависит от набора твоих подпроектов. Типовой шаблон:

| Содержимое файла | Куда |
|------------------|------|
| Клиенты, CRM, сделки, NDA, договоры | `<work-project>/INBOX/` |
| Сообщество/мероприятия | `<community-project>/INBOX/` |
| Личное, финансы, здоровье | `<personal-project>/INBOX/` |
| CV, био, визитки | `<personal-project>/Brand/` |
| Telegram экспорты, архивы чатов | `<personal-project>/INBOX/` |

Если в workspace только один проект — убери таблицу и перемещай прямо в папку проекта.

### Step 3: Прочитать файл перед перемещением

**ОБЯЗАТЕЛЬНО** прочитай файл (хотя бы начало), чтобы правильно определить проект. Не полагайся только на название.

### Step 4: Переместить

- `mv` (не копировать)
- Если целевая папка не существует — создать
- Если файл относится к существующей сделке/проекту → сразу в `CRM/Active/[deal]/attachments/` или аналог

### Step 5: Обновить документацию (если применимо)

- Файлы для CRM-сделок → обновить `deal.md` + `changelog.md`
- Стратегические документы → обновить `Docs/Strategy/`

### Step 6: Проверить результат

```bash
ls INBOX/
```

Должен остаться только `README.md`.

---

## Типичные ошибки

- **Файлы обработаны, но не удалены из INBOX** — всегда используй `mv`, не `cp`
- **Резюме/CV отправлены в рабочий проект** — если это личные био → в личный проект
- **ChatExport не обработан** — телеграм-экспорты → в личный проект
