# LLM Wiki — кросс-проектная база доменных знаний

> Паттерн: [Karpathy LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) | [Реализация](https://github.com/Ss1024sS/LLM-wiki)

`<your-wiki-project>/wiki/` — знания об AI, инструментах, паттернах и фреймворках, которые питают все проекты. LLM строит и поддерживает wiki инкрементально.

## Маршрутизация: что куда идёт

| Информация | Куда | Пример |
|---|---|---|
| Доменные знания (AI, инструменты, паттерны) | `<your-wiki-project>/wiki/` | Паттерн LLM Wiki, обзор CMS |
| Знания из внешних источников для 2+ проектов | `<your-wiki-project>/wiki/` | Статья про design systems |
| Концепции, мудрость, self-management | `<your-wiki-project>/wiki/` | Энергосистема, дисциплина |
| Шаблоны интеграций и инструментов | `<your-knowledge-base>/templates/` | Todoist-промпт |
| Контакты, нетворкинг | `<your-knowledge-base>/networks/` | Профили участников |
| Предпочтения, роль, фидбек для Claude | `memory/` | "пользователь предпочитает X" |
| Сделки, клиенты, встречи | `<your-agency>/CRM/` | Протокол встречи |
| Стратегия агентства | `<your-agency>/Docs/Strategy/` | Лидген-гипотезы |
| Личные цели, здоровье | `<your-knowledge-base>/goals/`, `Health/` | Месячные цели |
| Непонятно куда | `INBOX/` | Разобрать |

**Ключевой тест:** "Знание полезно 2+ проектам и не устареет через месяц?" → Wiki. Иначе → в проект.

## Структура

```
<your-wiki-project>/wiki/
├── index.md          ← каталог страниц (LLM читает первым)
├── current-focus.md  ← что сейчас изучаем (динамический)
├── sources.md        ← реестр обработанных источников
├── log.md            ← append-only таймлайн операций
├── raw/              ← неизменяемые сырые файлы
└── *.md              ← wiki-страницы (плоская структура)
```

## Операции

**Ingest** — пользователь даёт источник (статья, транскрипт, URL):
1. Сохрани сырой файл в `raw/` (или запиши URL в sources.md)
2. **Обсуждение (ОБЯЗАТЕЛЬНО):** прочитай источник → выдай 3-5 ключевых тезиса → спроси "что акцентировать?" Только после ответа — пиши страницы.
3. Создай/обнови wiki-страницу с frontmatter (включая `updated: YYYY-MM-DD`)
4. **Cross-references:** прочитай `index.md` → найди связанные страницы → добавь `## See also` с обеих сторон (обратная ссылка тоже). Один источник часто затрагивает 5-10 страниц.
5. Обнови `index.md` + `sources.md`
6. Обнови `current-focus.md` если тема новая
7. Append в `log.md`
8. **Счётчик lint:** проверь `log.md` — сколько ingest-записей с последнего lint. Если ≥ 10 → предложи запустить lint.

**Query** — вопрос по базе знаний:
1. Прочитай `index.md` → найди страницы → синтезируй ответ
2. **Формат вывода** — выбирай по типу вопроса:
   - Сравнение X и Y → таблица сравнения
   - "Как устроено / что такое" → markdown-страница
   - Временная динамика → хронология / список
   - Несколько независимых тем → структурированный список с заголовками
   - По умолчанию — текст с разделами
3. **Writeback (ВСЕГДА):** после любого нетривиального ответа — предложи сохранить результат как wiki-страницу. Не спрашивай "интересно ли тебе" — просто скажи "Сохранить это в вики как `<suggested-filename>.md`?"

**Lint** — запускается автоматически при счётчике ≥ 10 ingest ИЛИ по запросу. Чеклист:
- Orphan pages (нет входящих ссылок)
- Broken cross-refs (ссылки на несуществующие страницы)
- Stale claims (утверждения, опровергнутые более новыми источниками)
- Противоречия между страницами
- Важные концепции без собственной страницы
- Frontmatter drift (отсутствующие обязательные поля)
- Пробелы: темы, упомянутые но не раскрытые
После lint — append в `log.md` с результатами.

## Триггеры ingest (ОБЯЗАТЕЛЬНО)

**Когда пользователь говорит "изучи" / "в вики" / "ingest" + даёт URL, файл или текст** — это команда на ingest. **НЕ просто анализ в чат**, а полный воркфлоу (raw → страница → index → sources → log).

**После обработки транскрипта** (AI Traction, FFT, мастермайнды) — проверь, есть ли доменные знания для wiki. Если да — предложи ingest.

## Правила

- **raw/ неизменяем.** LLM не редактирует файлы в raw/.
- **index.md — точка входа.** LLM начинает работу с wiki через index.md.
- **log.md — append-only.** Не редактировать прошлые записи.
- **Frontmatter обязателен:** `title, source, created, updated, tags, status`.
- **`updated` обновляется при каждом редактировании.**
- **Cross-references двусторонние.** Добавил A→B? Проверь обратную B→A.
- **Хорошие ответы → в wiki.** Ценный синтез из query сохраняется как новая страница.
