# Skill: /write-post

> Написание поста для личного Telegram-канала автора с готовым форматированием

## Триггеры

- `/write-post`
- "напиши пост"
- "подготовь пост для телеги"
- "сделай драфт поста"

## Контекст

**Канал:** личный Telegram (настрой тематику под себя)
**Style Guide:** положи свой `style-guide.md` рядом и сошлись на него (опционально)

---

## Голос и стиль (обязательно)

### Основной принцип

**Пишу как думаю, а не как "надо".** Это личный канал с наблюдениями, а не корпоративный блог.

### Характеристики

| Есть | Нет |
|------|-----|
| Разговорный | Формальный |
| Личный опыт | Абстрактные теории |
| Наблюдения | Нравоучения |
| Размышления вслух | Готовые ответы |
| Простые слова | Канцелярит |

### Разговорные маркеры

Использовать естественные обороты:
- "Короче" — для перехода к сути
- "Прям" — для усиления
- "Ну и..." — для добавления мысли
- "В общем" — для подведения итога

### Чего НЕ делать

1. Не начинать с "Привет всем!" — сразу в суть
2. Не заканчивать на "Подписывайтесь!"
3. Не использовать много эмодзи — максимум 1-2 на пост
4. Не писать "экспертно" — делиться, а не учить
5. Не использовать таблицы в тексте поста
6. Не давать непрошеных советов

## Структура поста

```
1. Хук (личное наблюдение / история / провокация)
2. Развитие мысли (что заметил, почему это важно)
3. Конкретика (примеры, детали, цифры)
4. Рефлексия / вопрос к читателям
```

### Типы хуков

**Личная история:**
- "Столкнулся с интересной штукой..."
- "Последние полгода я..."

**Наблюдение:**
- "Заметил закономерность..."
- "Вижу как..."

**Провокация:**
- "«У нас 80% сотрудников используют ChatGPT!» — говорит компания. Поздравляю..."

### Концовки

**Вопрос к читателям:**
- "А у вас как?"
- "Что думаете?"
- "Сталкивались с таким?"

**Рефлексия:**
- "Пока не знаю, к чему это приведёт. Но наблюдаю."

## Форматирование — Telegram Markdown

### Синтаксис

```
**жирный**
*курсив* или _курсив_
~зачёркнутый~
`моноширинный`
||спойлер||
[текст ссылки](https://url.com)
```

### Правила

- **Жирный** — для ключевых понятий (не злоупотреблять)
- Списки: используй `—` или `•`
- Абзацы: короткие (2-4 предложения), одна мысль = один абзац
- Пустая строка между абзацами

### Ссылки — ОБЯЗАТЕЛЬНО

Если в тексте упоминаются исследования, источники, цифры — **делай гиперссылки**:

```
Исследование [HBR 2022](https://url) показывает...
По данным [Gloria Mark](https://url)...
Подробнее в [The Keyhole Effect](https://arxiv.org/abs/...)
```

**НЕ так:**
```
Исследование HBR 2022 показывает... (ссылка: https://...)
```

## Формат вывода — КРИТИЧНО

Готовый текст выдавать **внутри code block** (```), чтобы можно было скопировать одним кликом и сразу вставить в Telegram.

### Шаблон ответа

```
## Готовый пост для Telegram

\`\`\`
[Здесь текст поста с форматированием Telegram Markdown]
\`\`\`

## Заметки

- [технические заметки, что можно улучшить]
```

## Алгоритм работы

1. **Получи** тему/идею/материал от пользователя
2. **Прочитай** style-guide если нужно уточнить голос
3. **Напиши** пост по структуре: хук → развитие → конкретика → концовка
4. **Проверь:**
   - Звучит как разговор, а не как статья?
   - Есть гиперссылки на источники?
   - Нет таблиц, много эмодзи, канцелярита?
5. **Выдай** в code block с Telegram Markdown

## Пример

**Вход:** Идея про переключение контекста в CRM + исследование HBR про 1200 переключений

**Выход:**

```
Последние полгода я почти не открываю CRM.

Не потому что забросил продажи — сделок стало больше. Просто работаю с ними иначе.

Раньше: открыть Bitrix → найти карточку → вспомнить контекст → обновить поля → закрыть. И так [1,200 раз в день](https://www.atlassian.com/blog/loom/cost-of-context-switching) — реальная цифра из исследования HBR.

**4 часа в неделю** уходит только на то, чтобы "вернуться в контекст" после переключения.

Сейчас пишу в чат: "что там по сделке с Альфой?" — и получаю контекст. Никаких форм, кликов.

А вы как работаете с CRM?
```
