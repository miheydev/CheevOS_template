# Skill: /tov-write

> Генерация текста в заданном tone of voice по профилю

## Триггеры

- `/tov-write`
- "напиши в моём стиле"
- "напиши в стиле"
- "напиши голосом"
- "сгенери текст по профилю"

## Инструкция

Пользователь хочет сгенерировать текст, используя сохранённый ToV-профиль. Задача — написать текст, который звучит как автор профиля, а не как AI.

---

## Алгоритм

### 1. Определи профиль

**Если профиль указан явно** (имя файла, путь, или "в моём стиле"):
- Найди файл в `brand/tov-profiles/`
- Прочитай его

**Если профиль не указан:**
- Проверь `brand/tov-profiles/` — если там один профиль, используй его
- Если несколько — покажи список, спроси какой

**Если профилей нет:**
- Скажи: "Профилей пока нет. Используй `/tone-of-voice-analyzer` чтобы создать профиль из примеров текстов."

### 2. Определи задачу

Пользователь должен указать:
- **Что написать** (тема, идея, тезисы)
- **Формат** (пост, письмо, сообщение, статья) — если не указан, спроси
- **Длина** (опционально) — если не указана, определи по формату

### 3. Генерация

Загрузи профиль в рабочую память. При написании:

**ОБЯЗАТЕЛЬНО:**
- Используй лексику из профиля (характерные слова, обороты, маркеры)
- Соблюдай синтаксис (длина предложений, знаки препинания)
- Выдерживай ритм (абзацы, соотношение коротких/длинных фраз)
- Применяй авторские приёмы из профиля
- Начинай и заканчивай как указано в профиле
- Сохраняй эмоциональный регистр (шкалы)

**ЗАПРЕЩЕНО:**
- Использовать что-то из anti-patterns профиля
- Добавлять слова/обороты из списка "чего НЕ использовать"
- Менять регистр формальности (если профиль разговорный — не делать формальным)
- Вставлять "типичные AI-фразы" (безусловно, в заключение хочется отметить, и т.д.)

### 4. Самопроверка

Перед выдачей текста — пройдись по чеклисту:

- [ ] Звучит как автор, а не как AI?
- [ ] Характерные обороты присутствуют?
- [ ] Anti-patterns отсутствуют?
- [ ] Длина и структура соответствуют формату?
- [ ] Эмоциональный регистр выдержан?

Если хотя бы один пункт "нет" — перепиши.

## Формат вывода

Зависит от формата текста:

**Telegram-пост** → внутри code block (```) для копирования, с Telegram Markdown
**Письмо / сообщение** → внутри code block (```) для копирования
**Статья / документ** → сохрани в файл, в чат дай резюме
**Другое** → спроси у пользователя

## Совместная работа с другими скиллами

- `/write-post` — если у пользователя есть профиль для TG-постов, `/tov-write` может дополнять `/write-post`
- `/voice` и `/voice-tg` — для структурирования потока мыслей (не генерация, а упорядочивание)
- `/mega-polish` — для финальной полировки после генерации
