Пользователь просит зафиксировать урок из ошибки, которую ты допустил в этой сессии.

## Что делать

1. **Reflect** — опиши что конкретно пошло не так и почему. Не отмазывайся, будь честен.

2. **Abstract** — извлеки общий паттерн. Не "я неправильно отформатировал таблицу", а "я применил форматирование без проверки существующего стиля в файле".

3. **Generalize** — сформулируй правило, которое предотвратит эту и похожие ошибки.

4. **Маршрутизация** — определи, куда записать:

| Если... | Куда |
|---------|------|
| Критично, должно работать ВСЕГДА во всех проектах | **Предложи** правило в корневой `CLAUDE.md` (секция по теме). Жди подтверждения. |
| Критично, но только для одного проекта | **Предложи** правило в проектный `CLAUDE.md`. Жди подтверждения. |
| Контекстное, может устареть, предпочтение | Запиши в memory (тип `feedback`) |
| Мелкое, разовое, не повторится | Скажи вслух, не записывай никуда |

5. **Запись** — выполни запись по результатам маршрутизации. Для CLAUDE.md — формат правил из секции "Добавление новых правил". Для memory — frontmatter + Why + How to apply.

## Ограничения

- **НЕ** записывай в CLAUDE.md без подтверждения пользователя
- **НЕ** дублируй существующие правила — проверь перед записью
- **НЕ** записывай workaround'ы ("пропускай X для простых случаев")
- Будь кратким: правило = 1-2 строки, не эссе
