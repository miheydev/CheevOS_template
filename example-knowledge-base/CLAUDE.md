# example-knowledge-base — правила проекта

> Этот файл загружается автоматически, когда Claude Code работает внутри папки `example-knowledge-base/`. Дополняет корневой `CLAUDE.md`.

## Контекст

**Что это:** личная база знаний / second brain. Здесь живут цели, проекты, идеи, wiki, бренд-материалы.

**Кто пользователь в этом контексте:** владелец базы. Работает с личными целями, проектами, знаниями, контентом. Хочет, чтобы AI помогал организовывать, а не переорганизовывал.

## Структура (кратко)

- `Goals/` — цели (год → квартал → месяц)
- `Projects/Active/` — 1 папка = 1 личный проект
- `Ideas/` — сырые идеи, гипотезы
- `Wiki/` — LLM Wiki (доменные знания)
- `Brand/Content/` — ToV-профили, style guides, черновики
- `Learning/` — конспекты курсов, книг, видео

## Правила работы с Wiki

**Wiki — это не произвольная свалка заметок.** Это структурированная база, которая работает по паттерну [Karpathy LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f).

### Обязательные правила

- **`Wiki/raw/` неизменяем.** Никогда не редактируй файлы в raw — только добавляй новые
- **`Wiki/index.md` — точка входа.** Начинай работу с wiki с чтения index.md
- **`Wiki/log.md` — append-only.** Не редактируй прошлые записи, только добавляй новые
- **Frontmatter обязателен** на каждой wiki-странице: `title, source, created, updated, tags, status`
- **`updated` обновляется при каждом редактировании** — без этого не работает детектор устаревших страниц
- **Cross-references двусторонние.** Добавил ссылку A→B? Проверь обратную B→A

### Триггеры ingest

Когда пользователь говорит **"изучи"**, **"в вики"**, **"ingest"** + даёт URL, файл или текст — это команда на полный ingest-воркфлоу:

1. Сохрани сырой файл в `Wiki/raw/` (или запиши URL в `sources.md`)
2. Создай/обнови wiki-страницу с frontmatter
3. Cross-references: прочитай `index.md` → найди связанные страницы → добавь `## See also` с обеих сторон
4. Обнови `index.md` + `sources.md`
5. Обнови `current-focus.md` если тема новая
6. Append в `log.md`

Полный ответ пользователю — в wiki-странице. В чат — только краткое резюме.

### Триггер writeback

Если ответ на вопрос пользователя содержит **нетривиальный синтез, сравнение или новый вывод** — предложи сохранить как новую wiki-страницу. Не сохраняй молча, но и не теряй ценный контент в чате.

## Правила работы с целями

- **НЕ** перезаписывай файлы целей целиком — только точечные правки через `Edit`
- При обновлении прогресса всегда добавляй дату: `- [x] <задача> (2026-04-10)`
- Если пользователь ставит сверх-амбициозную цель — **спроси** про реалистичность и промежуточные вехи, не молчи
- Цель без плана действий = надежда. Всегда уточняй: "а что конкретно будешь делать?"

## Правила работы с идеями

`Ideas/` — это **сырой слой**. Не применяй полировку, не структурируй без запроса. Пользователь хочет зафиксировать мысль быстро, детализация — потом.

Если идея превращается в проект — **переноси** (не копируй) в `Projects/Active/<slug>/` и заводи полную структуру.

## Правила работы с контентом и голосом автора

- При генерации любого поста/письма от имени пользователя — **обязательно** загрузи соответствующий ToV-профиль из `Brand/Content/tov-profiles/`
- Если профилей несколько — спроси, какой использовать
- Если профилей нет — не генерируй в стиле "эксперта" по умолчанию, спроси про голос и предложи сначала создать профиль через `tone-of-voice-analyzer`
- **Сохраняй разговорные обороты** пользователя — это НЕ филлеры, это живая речь

## Куда класть входящее

| Тип файла | Куда |
|-----------|------|
| Статья, видео, подкаст (для усвоения) | `INBOX/` → `Wiki/` через ingest |
| Скан документа (страховка, банк, медицина) | `INBOX/` → `Finance/` или `Health/` |
| Черновик поста / заметка для контента | `Brand/Content/Drafts/` |
| Случайная идея | `Ideas/` |
| Фотография тренировки | `Health/` (с датой в имени) |

## Чего НЕ делать

- **НЕ** создавай папки "на всякий случай" — пусть будут, когда понадобятся
- **НЕ** копируй личные файлы в `example-agency/` — это разные контексты (личное vs работа)
- **НЕ** смешивай идеи и проекты — идея сырая, проект с дедлайном и планом
- **НЕ** создавай wiki-страницы без прочтения `index.md` — гарантированно получишь дубликат
