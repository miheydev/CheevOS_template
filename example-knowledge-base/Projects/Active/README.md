# Projects/Active/

Активные личные проекты. **1 папка = 1 проект.**

> Не путать с рабочими сделками / клиентскими проектами — те живут в `../../example-agency/CRM/Active/`.
>
> Здесь — личные инициативы: запуск курса, написание книги, research-проект, хобби.

## Именование папки

```
<slug-проекта>/
```

Slug в kebab-case, без даты в имени (даты — внутри `<slug>.md`).

Пример: `personal-ai-os/`, `book-on-llm-wiki/`, `marathon-prep-2026/`

## Структура папки проекта

```
<slug>/
├── <slug>.md         # Главный файл — описание, цели, статус, следующие шаги
├── changelog.md      # Хронология изменений (append-only)
└── attachments/      # Файлы, ресёрч, скрины
```

## Шаблон \<slug\>.md

```markdown
# <Название проекта>

**Статус:** 🟢 Активный / 🟡 Пауза / ⚪ Архив
**Старт:** YYYY-MM-DD
**Дедлайн:** YYYY-MM-DD (если есть)

## Зачем

Почему я это делаю. 1-3 абзаца. Если не можешь ответить — проект скорее всего не нужен.

## Как измерить успех

Конкретные метрики или признаки "сделано".

## Этапы

- [ ] Этап 1 — <результат>
- [ ] Этап 2 — <результат>

## Следующие шаги

1. <конкретное действие> — <когда>
2. ...

## Риски и блокеры

- <что может помешать>

## Связанное

- Wiki-страницы: [wiki-page.md](../../../Wiki/wiki-page.md)
- Идеи-источники: [../../Ideas/idea.md](../../Ideas/idea.md)
```
